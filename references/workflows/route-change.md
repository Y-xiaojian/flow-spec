# Route B 修改业务（文档侧）

1. 头脑风暴（可选，输出 `changes/<CHANGE-ID>/brainstorm/`）
2. 任务计划（输出 `changes/<CHANGE-ID>/tasks/`）
3. 影响分析（定位受影响需求、流程图、技术说明）
4. 需求修订（必要时）
5. 流程图 / 技术说明修订（必要时）
6. 一致性检查（术语、编号、交叉引用）
7. 输出到 `changes/<CHANGE-ID>/…` 并记录日志

规则：

- 范围变化先改需求或记录风险，再改流程图与技术说明。
- 所有改动必须追加日志到 **`{产出根}/logs/`**（见 `storage.md`）。
