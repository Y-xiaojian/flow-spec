# 规范与约束

## 通用硬约束

1. 技术设计不得超出已确认需求或规格范围。  
2. 推断内容必须标注 `inferred`。  
3. 每次执行应记录输入、输出、路径与结果（见存储与日志约定）。  
4. 产物写入输出根目录下对应子目录（默认 `flow-spec/temp/`）。  
5. 若项目提供术语表或名词表，输出前应核对一致性。  
6. 每条选定路线应落地头脑风暴或等效设计留痕（若该路线要求）。  
7. 每条路线应落地任务清单（若该路线要求）。  

## 追溯约束

- 需求编号：`FT-xxx`（若项目采用其它前缀，以项目为准）  
- 校验规则：`VR-xxx`  
- 业务规则：`BR-xxx`  
- 技术文档宜含需求追溯表  

## 文档质量

- 需求文档结构以 `requirement-doc-writing` 技能及关联模板为准。  
- 技术文档宜含流程图与时序图。  
- 若产出含验收或评审记录，宜含结论与修订依据（按项目约定）。
