# 原型交付物通用模板（Prototype Spec）

> **性质**：可复制粘贴的结构模板与字段约定，**不包含**具体颜色或像素。视觉与 CSS 以项目《前端样式规范》为准。

---

## 1. 与另两份文档的关系

| 文档 | 作用 |
|------|------|
| `prototype-guidelines.md` | 流程、类型、数据与标注原则 |
| **本文件** | 页面清单、标注表、可选 HTML 结构清单的**填空模板** |
| 《前端样式规范》 | 视觉与组件实现细则（产品可替换） |

---

## 2. 页面清单模板（CHECKPOINT 前输出）

在生成 `index.html` 之前，先输出以下结构供确认：

```markdown
## 页面清单
| 页面 key | 页面名称 | 类型 | 核心内容 | 上游入口 |
|----------|----------|------|----------|----------|
| list | 询价单列表 | 列表 | 筛选、多状态行、新建 | 侧栏 |
| ... | ... | 列表/表单/详情/审批/报表 | ... | ... |

## 页面跳转
- list --[新建]--> create
- ...
```

- **页面 key**：英文小写短横线，与脚本中 `showPage('key')` 一致（若样式规范约定其他路由方式从其约定）。

## 3. `annotations.md` 结构模板

每个在 HTML 中出现的逻辑页面（或独立弹窗流程）一节：

```markdown
## <页面名称>（key: <pageKey>）

| 元素说明 | 控件类型 | 规则/业务约束 | 接口或字段建议名 |
|----------|----------|----------------|------------------|
| 询价标题 | input | 必填，长度限制见 PRD | title |
| ... | ... | ... | ... |
```

可选追加：

```markdown
### 状态与权限
- 角色 A：可见… / 不可见…
```

---

## 4. 单页多视图 HTML 的推荐结构（逻辑模板）

以下为**逻辑块**顺序建议，类名与 DOM 细节以《前端样式规范》为准。

1. **布局骨架**：全局导航区 + 顶栏 + 内容区（具体分栏比例见样式规范）。  
2. **多页面容器**：每个业务页面一个根节点，`id="page-<key>"`，默认仅一个带「当前可见」状态类。  
3. **弹窗**：独立 `modal-mask` 节点，`id` 与 `showModal` / `closeModal` 参数对应。  
4. **脚本**：提供与样式规范一致的**页面切换**、**弹窗开关**函数名（本仓库默认 `showPage` / `showModal` / `closeModal`）；若规范改名，全页统一替换。

## 5. 模拟数据检查清单（生成后自检）

- [ ] 列表：至少 4 条且状态不全相同  
- [ ] 表单：主要字段有预填  
- [ ] 图表区：有 `chart-placeholder` 或等价占位，且**有文字说明**（若样式规范要求）  
- [ ] 关键操作有 `data-annotation`（或项目规定属性）
