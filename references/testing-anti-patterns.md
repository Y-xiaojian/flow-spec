# 测试反模式（摘录）

在编写或修改测试、增加 mock 时阅读本节。

**原则：** 测真实行为，不测 mock 行为。mock 只为隔离依赖，不是被测对象。

**铁律：**
1. 不测 mock 的行为  
2. 不给生产类加「仅测试用」的公开方法  
3. 不理解依赖就不要乱 mock  

## 反模式：测了 mock

错误：断言「mock 被调用」或「页面上有 mock 组件」，通过与否与真实逻辑无关。

正确：断言对用户可观测的结果、返回值或与真实依赖交互后的状态。

## 反模式：测试专用 API

为测试方便给生产类增加 `resetForTest()` 等——破坏封装。应通过可注入依赖、工厂或测试替身在测试层解决。
