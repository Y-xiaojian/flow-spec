# 飞书画板 · 泳道流程图（跨职能 SOP 样式）

本技能描述**飞书（Lark）画板**上可用的版式与语义约定，使流程图与示例「离职流程图」一类稿件**视觉一致、节点可读、长文下沉脚注**。不绑定具体菜单路径（产品迭代会变）；绘图时在飞书中选用等价图形与样式即可。

## 何时使用

- 需在飞书画板绘制 **纵向泳道**、**部门/角色分列** 的流程图；
- 节点内需 **简短标题**，细则/SOP 条文放在 **底部说明区**；
- 需在节点角标 **❶❷❸❹**（或 ①②③④）与脚注 **一一对应**。

## 版式总览

| 要素 | 约定 |
|------|------|
| 画布方向 | 泳道 **横向并列**（左→右为角色），流程 **自上而下** |
| 标题 | 画布 **顶部居中**，黑体，如「离职流程图」「转岗流程图」 |
| 泳道列数 | 常用 **4 列**（可按流程增减），列宽约均等 |
| 脚注区 | 画布 **底部** 独立矩形区域，标题 **「【交接说明】」**，与主流程区留白分隔 |

## 泳道（Swimlane）

- **结构**：每列一条泳道；**泳道名**写在列顶标题区（与示例一致：技术支持工程师 / 技术支持部助理 / 技术支持部主管 / 人力资源部）。
- **顺序**：从左到右一般为 **执行者（员工）→ 支撑岗位 → 审批线 → HR/系统**，最终以业务确认的 RACI 为准。
- **对齐**：同泳道内步骤 **垂直居中对齐列宽**；节点距泳道左右边距建议 **≥24px**，避免贴边。

## 图形与配色（与示例对齐）

**填充与描边（可作参考色值）**

| 用途 | 填充 | 描边/强调 |
|------|------|-----------|
| 起止节点（胶囊） | 浅灰 `#F5F5F5` | 深灰描边 |
| 常规流程矩形（圆角） | 浅蓝 `#E1F5FE`（或同类浅蓝） | 深蓝描边 |
| 起始第一步（可选强调） | 白底 | 灰色描边（与示例首开节点一致） |
| 决策菱形 | 浅橙 `#FFF3E0` | 橙色描边 |
| 脚注框 | 浅灰底或白底 | 浅灰虚线或细实线 |

**形状语义**

- **起止**：横向圆角矩形（胶囊形）。
- **步骤**：圆角矩形；文案单行优先，必要时两行。
- **分支**：菱形；出口箭头标注条件（如「交接完成」「未交接完成」）。
- **连线**：细灰箭头；主路径保持一致线宽。

## 脚注系统（核心）

1. **编号角标**：需要展开的步骤，在矩形 **左上角** 放置小号编号徽章 **❶ ❷ ❸ ❹**（白字深色圆底或与飞书自带贴纸一致即可）。
2. **节点内文案**：只写 **精简动作名**（8～16 字为宜），不写完整 SOP 条文。
3. **底部【交接说明】**：按编号逐项写 **完整说明**，须包含文档编号（如 `QT-SOP-FAE-380-01`）、系统名（如 QHR）、例外规则等。
4. **一致性**：图中出现的每一个 ❶❷❸❹，脚注区必须有对应条目；脚注条目勿重复使用未出现的编号。

### 示例映射（与示例图一致的可读写法）

| 角标 | 节点短文案（示例） | 脚注长文案方向 |
|------|-------------------|----------------|
| ❶ | 指定交接人并监督移交签字 | 指定交接人、跟进待办/项目/资产、监督执行、移交完成签字 |
| ❷ | 依清单办理交接手续 | 依据 `QT-SOP-FAE-380-01` … 办理部门内交接 |
| ❸ | 依清单核对交接结果 | 依据同一清单核实是否交接完成 |
| ❹ | QHR 自动发起离职交接并通知交接人 | 系统自动发起流程、通知各部门交接人等 |

（转岗场景将「离职」改为「转岗」并保持编号脚注同步替换。）

## 文案规则

- **节点**：动词开头或「名词+动词」短句；避免一段话塞进矩形。
- **连线标签**：分支写 **结果状态**（完成/未完成、同意/驳回），不写长理由。
- **脚注**：允许完整句与文档引用；编号列表可用 **①②③④** 或 **1）2）3）**。

## 交付前自检

- [ ] 四类形状语义正确（起止 / 步骤 / 决策 / 脚注框）
- [ ] 配色未混杂过多色相（以浅蓝步骤 + 橙色决策 + 灰脚注为主）
- [ ] 每个角标在【交接说明】中有条目；条目编号与图中一致
- [ ] 决策出口两条以上时有清晰标签
- [ ] 打印或缩放预览时正文 ≥11pt 等效字号可读

## 与 Flow-Spec `diagram-skill` 的关系

- **`diagram-skill`**：产出 **draw.io（.drawio）**（YAML→CLI→draw.io），适合入库与文档引用。
- **本技能**：产出在 **飞书画板** 上的**手绘等价规范**（样式与脚注逻辑对齐）；若需归档源文件，可截图插入文档或导出图片后附路径说明。
